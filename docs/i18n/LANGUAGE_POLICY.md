# Language Policy

Supported languages:
- ar
- en
- fr
- it
- tzm

Rules:
1. Every new UI label must use a translation key
2. No hardcoded business labels inside pages when avoidable
3. Shared labels should live in locale JSON files
4. Web and App must consume the same translation keys when possible
5. API business values should remain stable and language-neutral
6. Human-readable labels should be translated in UI layer
